# S2-057: Flow Guide (Layer 1)

**Epic:** EPIC-7 — Integration Documentation + Manual QA

---

## Мета

Користувач API розуміє загальну картину: навіщо цей інструмент, які основні кроки, як вони пов'язані між собою. Це "карта" — не інструкція для виконання, а орієнтир.

## Що створюємо

`docs/api/flow-guide.md` — одна сторінка з:

1. **Призначення системи** — що робить course-supporter (трансформація навчальних матеріалів у структурований план курсу)
2. **Основний flow** — послідовність кроків:
   - Автентифікація (API key)
   - Створення курсу
   - Побудова дерева матеріалів (nodes)
   - Завантаження матеріалів (файли/URL)
   - (Опціонально) Налаштування slide-video mappings
   - Генерація структури курсу (free / guided mode)
   - Отримання результату
   - Моніторинг (jobs, cost reports)
3. **Діаграма flow** — текстова або mermaid-схема
4. **Типи матеріалів** — що підтримується (video, presentation, text, web) і як обробляється
5. **Async pattern** — пояснення що операції асинхронні, як працює polling через jobs

## Як робимо

1. Вивчити реальні API routes, schemas, flow
2. Написати текст від загального до конкретного
3. Узгодити з автором проєкту
4. Не включати curl-приклади (це Layer 2) — тільки концептуальний опис

## Очікуваний результат

Людина, яка прочитала Flow Guide, може відповісти:
- Що робить ця система?
- Які кроки потрібні від початку до результату?
- Які формати матеріалів підтримуються?
- Як працюють асинхронні операції?
