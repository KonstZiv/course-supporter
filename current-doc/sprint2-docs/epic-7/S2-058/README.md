# S2-058: Quick Start (Layer 2)

**Epic:** EPIC-7 — Integration Documentation + Manual QA

---

## Мета

Користувач API може пройти один конкретний найпростіший шлях від початку до результату, копіюючи curl-команди з документації. Кожен крок протестований на production.

## Що створюємо

`docs/api/quick-start.md` — покрокова інструкція з curl-прикладами:

1. Автентифікація — перший запит з API key
2. Створення курсу
3. Створення кореневого node
4. Завантаження одного текстового матеріалу (найпростіший формат)
5. Polling job status до завершення
6. Перевірка що матеріал оброблений (state: ready)
7. Генерація структури (free mode)
8. Polling generation job
9. Отримання згенерованої структури

## Як робимо — ітеративний процес

Для **кожного кроку**:

1. Написати curl-приклад
2. **Виконати його на `api.pythoncourse.me`** (реальний запит)
3. Перевірити response — чи відповідає очікуванню
4. Якщо знайдено баг:
   - Зафіксувати проблему
   - Виправити код
   - Написати unit test для бага
   - Задеплоїти фікс
   - Повторити запит
5. Оновити документацію з реальним response
6. Перейти до наступного кроку

Після проходження всіх кроків — **повторний прохід від початку до кінця** (smoke test всього flow).

### Варіації для тестування

Після основного happy path — протестувати той самий flow з різними типами матеріалів:
- Text: .md файл (маленький), .docx (великий)
- Presentation: .pdf, .pptx
- Video: .mp4 (коротке)
- Web: URL на реальну статтю

Результати тестування варіацій фіксуються, але в Quick Start залишається **тільки один найпростіший шлях**.

## Очікуваний результат

- Документ з 9 кроками, кожен з curl-прикладом і очікуваним response
- Всі приклади працюють copy-paste на production
- Знайдені баги виправлені і покриті тестами

## Тестування

**Автоматизовано:** `mkdocs build --strict`

**Manual QA:** Кожен curl-приклад виконаний на production. Повний flow пройдений end-to-end.
